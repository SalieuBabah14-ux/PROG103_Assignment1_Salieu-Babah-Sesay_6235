
RATIONALE
Student Record Management System

A) PROBLEM DESCRIPTION
In many educational institutions, student records are still managed manually using paper-based systems or simple spreadsheets. These methods are inefficient, time-consuming, and prone to errors such as data loss, duplication, and difficulty in retrieving information.
For example, searching for a student’s record in a manual system may take a long time, especially when dealing with large amounts of data. Additionally, updating or deleting records can be difficult and may lead to inconsistencies.
Therefore, there is a need for an automated system that can efficiently manage student records, improve accuracy, and reduce the workload on administrators.
B) SYSTEM EXPLANATION
The Student Record Management System is a simple console-based application developed using Python. The system allows users to manage student data effectively by performing the following operations:
	Add new student records 
	View all student records 
	Search for a specific student 
	Update student information 
	Delete student records 
Each student is identified using a unique registration number, and their details such as name, age, and course are stored in the system.
The system uses a menu-driven interface that allows users to interact with it easily. It continuously runs until the user chooses to exit, ensuring smooth and continuous operation.
C) STRUCTURED PROGRAMMING APPLICATION
The system is developed using structured programming principles, which focus on organizing code into clear, logical, and reusable components.
The following structured programming techniques were applied:
	Modular Programming: The program is divided into functions such as add_student(), view_students(), search_student(), etc. 
	Sequence: Instructions are executed step by step in a logical order. 
	Selection (Decision Making): if/else statements are used to make decisions based on user input. 
	Iteration (Looping): A while loop is used to repeatedly display the menu until the user exits. 
This approach makes the program easy to read, debug, and maintain.

3. HOW STRUCTURED PROGRAMMING PRINCIPLES WERE APPLIED
Structured programming principles were applied in the development of this system in several ways:
1.	Modularity
The program is divided into separate functions, each responsible for a specific task such as adding, viewing, updating, or deleting records. This improves code organization and reusability. 
2.	Control Structures 
	Sequence: Instructions follow a logical order from start to finish. 
	Selection: if/else statements are used to handle user choices and validate inputs. 
	Iteration: A while loop ensures the program continues running until the user exits. 
3.	Data Structures
A dictionary is used to store student records, allowing efficient data storage and retrieval using unique registration numbers. 
4.	User Interaction
The system uses input and output functions to interact with the user, ensuring that operations are performed based on user commands. 
5.	Code Readability and Maintainability
The use of clear function names, proper indentation, and logical structure makes the program easy to understand, debug, and extend.
